# Contributing to CreamCollar – AUTOSAR ARXML Code Generation

Thank you for your interest in contributing to this project!
We welcome **all types of contributions**-bug reports, feature requests, documentation improvements, and code enhancements.
Please follow the guidelines below to ensure smooth and effective collaboration.

---

## Reporting Bugs?

If you find a bug, please follow these steps:

1. **Check existing issues**
   Before creating a new issue, search the GitHub Issues page to ensure the bug hasn’t already been reported: [Issues](https://github.com/Creamcollar/autosar-arxml-codegen/issues)

2. **Create a new issue (if needed)**
   If the issue does not already exist, feel free to open a new one. Please include:

   * A clear and descriptive title
   * Detailed steps to reproduce the issue
   * Expected vs actual behavior
   * Relevant logs, screenshots, or error messages
   * A minimal code sample or test case (if applicable)

Well-documented issues help us resolve problems faster 

---

## Getting Started

To start contributing:

1. **Fork** the repository to your GitHub account.
2. **Clone** your fork locally:

   ```bash
   git clone https://github.com/<your-username>/autosar-arxml-codegen.git
   ```
3. Navigate to the project directory:

   ```bash
   cd autosar-arxml-codegen
   ```
4. Create a new branch for your work:

   ```bash
   git checkout -b feature/your-feature-name
   ```

   or

   ```bash
   git checkout -b bugfix/your-bug-fix
   ```

---

## Code Style & Guidelines

Please follow the existing project structure and adhere to the current coding conventions. Ensure that the code remains clean, easy to read, and well-commented where necessary. Avoid making unrelated changes within the same commit to keep the history clear and manageable. If your changes affect how the code is used or how it behaves, be sure to update or add the relevant documentation accordingly.

---

## Submitting Changes

When your work is ready:

1. Commit your changes with **clear and meaningful commit messages**:

   ```bash
   git commit -m "Fix: Correct ARXML parsing for Dio module"
   ```
2. Push your branch to your fork:

   ```bash
   git push origin feature/your-feature-name
   ```
3. Open a **Pull Request (PR)** against the `main` branch of the original repository.
4. In the PR description, clearly explain:

   * What problem your change solves
   * What approach you used
   * Any limitations or follow-up work (if applicable)
